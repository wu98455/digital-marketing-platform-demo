# 原型指南 · 从新建项目到 GitHub 部署（Pro v6）

> **给 Cursor / AI 的说明（必读）：**
>
> 当用户说「按原型指南做」「按原型指南新建原型」「新建客户原型」「部署 demo」时：
>
> 1. **严格按本文档阶段顺序**执行，每阶段完成检查清单后再进入下一阶段。
> 2. **阶段 1.2 必须由 AI 自动执行，不要交给用户手动做：** 从母版 `xcopy` 复制完成后，立即在项目目录执行：
>    ```powershell
>    Remove-Item -Recurse -Force .git
>    git init
>    ```
>    除非用户明确说「保留 git 历史」，否则**禁止跳过**此步骤。
> 3. 新项目路径优先放在 `D:\dev\projects\` 下。
> 4. **禁止**在母版 `antd-prototype-starter` 上直接改客户业务。
> 5. **命名采用方案 A（默认）：** 本地文件夹用**中文**，GitHub 仓库 / Demo 网址用**英文**。复制并完成 `git init` 后，**AI 必须根据文件夹名称生成 `项目对照表.md`**（见阶段 1.3），后续配置以对照表为准。
> 6. **本地优先，部署由用户控制：** 日常改原型时，AI 在本地改完并启动 `pnpm start` 供浏览器验收；**禁止**每改一点就 `git push`。仅当用户明确说「推到 GitHub / 部署一波 / 可以上线」时才 push 部署。

本文档随模板一起复制到每个新项目。每个客户 / 每个演示原型走一遍本流程即可。

**母版仓库（仅作复制来源，不要直接改业务）：**

- 本地：`D:\dev\projects\antd-prototype-starter`
- GitHub：https://github.com/wu98455/antd-prototype-starter
- Demo：https://wu98455.github.io/antd-prototype-starter/
- 技术栈：官方 **Ant Design Pro v6 simple**（Umi Max 4）

**重要原则：**

| 原则 | 说明 |
|------|------|
| 1 项目 = 1 文件夹 = 1 GitHub 仓库 = 1 Demo 链接 | 不要把多个客户塞进同一个仓库 |
| 从模板复制 | 不要空文件夹从零搭 ProLayout / 登录 / 部署 |
| Mock 数据 | 演示阶段不依赖后端 |
| **本地优先，推送由用户控制** | 当天在本地连续改 + `pnpm start` 预览；**只有用户明确要求时**才 push 部署 Pages；禁止改一点推一点 |
| push 需代理 | 国内 push GitHub 需开代理（端口一般为 **7892**） |

### 命名方案 A（默认：本地中文 + 部署英文）

| 用途 | 用什么 | 示例 |
|------|--------|------|
| **本地文件夹** | 中文（你手动复制时改名） | `示例项目` |
| **系统页面标题** | 中文 | `示例项目演示系统` |
| **GitHub 仓库名** | 英文小写 + 连字符 | `demo-project` |
| **Demo 链接** | 英文（与仓库名一致） | `https://wu98455.github.io/demo-project/` |

本地路径示例：`D:\dev\projects\示例项目`

**英文仓库名生成规则（AI 自动）：** 将中文文件夹名转拼音、小写、连字符连接，末尾加 `-demo`。  
例：`示例项目` → `demo-project`。若用户指定英文名，以用户为准。

**注意：** `PUBLIC_PATH`、`package.json` 的 `name`、GitHub 仓库名均用**英文仓库名**，不要用中文文件夹名。

## 阶段 5：修改部署配置

### 5.1 修改 `package.json` 的 build:pages


### 5.2 修改 `.github/workflows/deploy-pages.yml`

```yaml
env:
  PUBLIC_PATH: /demo-project/
run: pnpm run build:pages
```

`config/config.ts` 已读取 `process.env.PUBLIC_PATH`，一般无需改源码。

Umi 已启用 `exportStatic: {}`，适合 GitHub Pages 静态部署。
